## ⚠️ 給 Claude Code 的工作規則（重要，請遵守）

1. **不要主動加新功能**。除非用戶明確要求加什麼。這個專案的核心原則是「先驗證再加功能」。
1. **改 sig 字典時一律用 `.get()` 容錯**，不要直接索引 `sig["xxx"]`，避免缺字段時崩潰。
1. **任何代碼改動前**，先告訴用戶會動到哪些檔案、哪些函式，**等用戶確認再動手**。不要擅自連續改一堆。
1. **改完一定先驗證**：跑 `python -m py_compile analyzer.py bot.py` 確認沒語法錯誤。
1. **不要動 entry_grade 的內部值**（S/A/B/C/D），全系統有 7 處依賴它做比較判斷。只在顯示層用 `entry_grade_display()` 翻譯成「高品質/一般品質/低品質」。
1. **不要動結算權重**（TP1=15%、TP2=35%、TP3=35%、TP4=15%）。歷史 SIGNAL_RESULTS 依賴此設定，改了會讓舊數據不可比。
1. **重要原則**：用戶傾向不斷想加新東西（新聞權重、自動下單、App、訂閱）。**策略還沒被數據證明是正期望之前，什麼都別加**。如果用戶提出加新功能，可以禮貌地提醒這個原則，但最終由用戶決定。

## 一、這是什麼

一個 Telegram 加密貨幣交易信號機器人（頻道名「黑潮船長 / Signals BOT」）。

- 自動掃描多個幣種、用技術指標算出進場信號、推播到 Telegram。
- **只推信號給人看，不自動下單**（下單目前是人工去 BingX 點）。
- 作者在台灣，用繁體中文溝通，主要用手機操作。

## 二、技術架構

- **語言/檔案**：Python。核心兩個檔：
  - `analyzer.py`（約 7000 行）：策略、指標計算、信號生成、新聞分析模組
  - `bot.py`（約 3300 行）：Telegram 介面、推播、結算、數據持久化
  - `requirements.txt`、`.gitignore`
- **開發環境**：GitHub Codespaces
- **部署**：Railway，service 名叫 `worker`，從 GitHub `main` 分支自動部署
  - 部署方式：改 code → `git add . && git commit -m "..." && git push` → Railway 自動部署
- **數據持久化**：Upstash Redis（雲端，重新部署不會丟）。環境變數：
  - `UPSTASH_REDIS_REST_URL`、`UPSTASH_REDIS_REST_TOKEN`
  - 沒設會退回 /tmp 本地檔（重啟會丟）
- **其他環境變數**：
  - `TELEGRAM_BOT_TOKEN`（Telegram bot 的 key）
  - `ANTHROPIC_API_KEY`（新聞分析用，可選，沒設新聞功能靜默關閉）
  - `CRYPTOPANIC_TOKEN`（新聞來源，可選）

## 三、當前版本：v55

### 策略邏輯（開單三道關卡）

1. **7+1 策略投票（strategy_consensus）**：
- 7 個技術策略：趨勢追隨、動量、量價、均線、支撐阻力、BOS 突破、訂單流
- 第 8 票：新聞情緒（看多/看空且強度≥3 時加同方向票）
1. **五維評分（滿分 100）**：趨勢+動量+結構+量能+風險。
1. **進場品質分級 + tier 分級**：
- tier = S/A/B/C（推播分級）
- entry_grade = S/A/B/C/D（進場時機品質）—— 顯示時翻譯成中文：S/A→高品質, B/C→一般品質, D→低品質。**內部值絕不能改**。

- 最後還要過：盈虧比門檻（TP1≥1.2R）、冷卻、保護模式、大盤閘門。

### 重要設計細節（容易踩雷）

- **兩套評級會疊加**（tier 和 entry_grade 是不同東西）。
- **win_rate 是公式推算值**（×0.82 校準），不是真實回測。真實樣本≥10 會用真實值覆蓋。
- **結算 final_pct 用分批止盈加權**：TP1=15%、TP2=35%、TP3=35%、TP4=15% 倉位權重。「帳面浮盈」(全倉現價) 和「歷史結算值」(分批加權) 會不同，這是正常的。
- **緊急保底單**：盤整期硬擠的低質觀察單（score 26、進場品質D）。已加嚴觸發條件（ADX>15 + RSI>60/<40）。

### v53~v55 做過的重要修復（別重複改）

## 六、核心指導原則

1. **先驗證，再加功能。** 策略沒被數據證明是正期望前，什麼都別加。
1. **看期望值，不要看勝率。** 33% 勝率配正賠率可能是賺的；80% 勝率配爛賠率可能是賠的。
1. **穩 ≠ 賺。** 小樣本是雜訊。誠實面對數據，不美化。
1. **連敗是必然。** 任何正期望策略都會連敗，追求「無連敗」會害死策略。
1. **指標不能完美預測。** bot 的價值是紀律 + 概率篩選，不是算命。
