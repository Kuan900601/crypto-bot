## 1. 策略與鐵則

- **只換前端視覺外殼，完全保留後端 / 資料層 / 付費（NOWPayments）/ 認證（NextAuth、tier）邏輯。** 資料怎麼來、權限怎麼擋，維持現狀。
- **不要動** bot.py / analyzer.py / 任何策略 / 金流 webhook / tier 計算（tierOf）/ API 授權 / sanitizeSignal 授權邏輯。
- **全部在 `redesign` 分支做**（沒有就建一個並切過去）。過程中 **main（線上正式站）完全不動**。全部做完、驗證、看過完整 diff，才一次 merge 上線。
- **每頁做完跑** `npx tsc --noEmit` + `npm run build`，確認能編譯、能渲染、功能沒壞。
- **績效照實**：信號 / 回測 / 分析 / 首頁顯示的所有數字（勝率、報酬、回撤、損益）必須來自真實資料，有賺有賠照實，不挑漂亮的、不捏造美化。**真實達標樣本不足前，不准寫死任何績效數字。**

## 4. 設計與效能（摘要，細節見 design-system.md）

- 需 canvas/hooks/window 的頁面為 Client Component（'use client'）。
- 響應式顯示/隱藏一律 CSS `@media`，**禁止 render 時用 window.innerWidth 判斷**（內建瀏覽器破圖閃爍）。
- transform 動畫加 `will-change:transform`；Canvas 手機降載（dpr≤1.4、粒子減半、visibilitychange 暫停）；支援 prefers-reduced-motion。
- 手機（375–414px）每頁不破版、標題行高≥1.15 不被切。
- favicon / 社群 / PWA 指向 `public/brand/blacktide-logo-1024.png`，更新 metadata 與 manifest。Vercel region hnd1 維持。

## 5. 首頁（重點，含新轉換文案）

### 5.1 結構
採用 landing-v8.jsx 完整結構：Hero（燈塔+海浪+漂浮信號卡）→ 今日 AI 信號預覽卡 → 近 30 天績效區 → 行情跑馬燈 → 信號戰績流 → 底部震撼轉換區 → 頁尾。改寫成 Next.js 結構，資料沿用 /api/signals、/api/market。

### 5.2 文案與 CTA（照這個改）
- **Hero 主標**：AI 24 小時盯盤，幫你抓加密貨幣交易信號
- **Hero 副標**：登入即可查看今日 AI 分析、方向判斷與部分交易信號。升級 Pro 解鎖進場、止損與 TP 點位。
- **所有主要 CTA 統一**：免費註冊 · 查看今日信號
- **「免費上船 / 上船 / 送 3 日 Plus」等舊文案全部換成**：免費註冊 · 查看今日信號
- **未登入狀態的「升級解鎖」按鈕改成**：登入解鎖完整信號
- **Navbar 右上固定顯示「免費註冊」**（不要只顯示「登入」）。
- **統計數字改成真實的**（移除假會員數字 Plus 284 / Pro 57 / 即時在線 593）：監測幣種 52 / 今日信號 7 / 24/7 AI 盯盤。
- **5 秒看懂目標**：訪客一進來要立刻看懂「這裡能看方向、信號、進場、止損、TP」。

### 5.3 今日 AI 信號預覽卡
版面：幣種 + 方向 + 信心，下方鎖住進場/止損/TP，底部 CTA。
- 幣種、方向：**接 /api/signals 的真實當前信號**（例如 BTC/USDT 做空），不要寫死。
- 信心：用模型的 **AI 信心 / 模型評分**，並標明「模型評分」不是「歷史勝率」。**不要寫死 72% 之類的勝率數字。**
- 進場價 / 止損價 / TP1：顯示「🔒 登入解鎖」（依現有付費牆，未登入鎖住）。
- 按鈕：免費註冊 · 查看完整信號

### 5.4 近 30 天績效區
- **只有接真實 SIGNAL_RESULTS 才顯示數字。** 顯示勝率 / 平均報酬 / 最大回撤時，**用真實值並標註樣本數**（例如「近 N 筆」）。
- 若真實樣本太少或數據不漂亮，**先不要放績效數字區**，改放過程價值：52 幣種監測、7+1 策略投票、分批止盈紀律、24/7 AI 盯盤。
- **絕對不要寫死 68% / +12.4% / -6.8% 之類未經真實資料支持的數字**（台灣投顧法對績效廣告管制嚴，誤導性績效展示有法律風險）。

### 5.5 行情跑馬燈
接 /api/market 真實幣價與漲跌幅。**移除假隨機跳動**（價格不能假跳），改真實值 + 每 30–60 秒刷新。

### 5.6 信號戰績流
接 /api/signals：已結算信號公開真實進場價+結果+損益%（當戰績，已結束非可操作）；進行中/最新鎖價（🔒）。**照實呈現輸贏，不挑漂亮的**；真實樣本不足就不強調損益數字。安全：只有「已結算」可公開價位，「進行中」依現有 tier 邏輯對未付費者隱藏。
